# Module 04 — Strings

---

# Why This Matters

Imagine a typical day as a DevOps Engineer.

You are constantly working with text:

* Server names
* IP addresses
* Log entries
* Error messages
* Environment names
* Docker image tags
* URLs
* API responses
* Configuration values

Almost everything humans read from a computer is text.

Python represents text using **strings**.

Without strings, it would be impossible to:

* Process log files
* Generate reports
* Build monitoring dashboards
* Create deployment messages
* Read configuration files
* Interact with APIs

Consider a monitoring system.

Instead of seeing:

```text
Server Status:
web01 → Healthy
```

you would see meaningless machine data.

Strings make software understandable to humans.

That is why strings are one of the most important data types in Python.

---

## Real DevOps Examples

Strings appear everywhere.

Server name:

```python
hostname = "web01"
```

Environment:

```python
environment = "production"
```

Docker image:

```python
image = "nginx:latest"
```

URL:

```python
api_url = "https://api.company.com"
```

Log message:

```python
log_message = "Deployment completed successfully"
```

Every one of these values is a string.

# Core Concepts

---

## What Is a String?

A string is a sequence of characters.

Examples:

```python
"hello"
```

```python
"web01"
```

```python
"production"
```

```python
"10.0.0.5"
```

All are strings.

---

## Why Strings Exist

Computers store data as numbers internally.

Humans communicate using text.

Strings bridge that gap.

They allow programs to work with human-readable information.

---

## Quotes Create Strings

Python recognizes text using quotation marks.

Single quotes:

```python
'web01'
```

Double quotes:

```python
"web01"
```

Both are valid.

---

## Recommended Style

For this handbook:

```python
"web01"
```

Use double quotes consistently.

This follows common Python practices.

# String Variables


# Checking Types


# Syntax

Creating strings:

```python

```

---

# String Concatenation

Concatenation means joining strings together.

```python
first = "web"
second = "01"

result = first + second

print(result)
```

Output:

```text
web01
```

---

## Spaces Matter

```python
first = "Hello"
second = "World"

print(first + second)
```

Output:

```text
HelloWorld
```

Notice the missing space.

---

Correct:

```python
print(first + " " + second)
```

Output:

```text
Hello World
```

---

# Repetition

Strings can be repeated.

```python
print("-" * 20)
```

Output:

```text
--------------------
```

Useful for reports.

# String Length

Use:

```python
len()
```

Example:

```python

```

```text

```

---

# Indexing

Each character has a position.

Example:

```text
web01

Position:

w  e  b  0  1
0  1  2  3  4
```

---

Access a character:

```python
hostname = "web01"

print(hostname[0])
```

Output:

```text
w
```

---

Another example:

```python
print(hostname[4])
```

Output:

```text
1
```

---

# Negative Indexing

Python can count from the end.

```text
web01

w  e  b  0  1
-5 -4 -3 -2 -1
```

Example:

```python
hostname = "web01"

print(hostname[-1])
```

Output:

```text
1
```

---

# Slicing

Extract part of a string.

Syntax:

```python
string[start:end]
```

Example:

```python
hostname = "web01"

print(hostname[0:3])
```

Output:

```text
web
```

---

Another example:

```python
ip = "10.0.0.5"

print(ip[0:2])
```

Output:

```text
10
```

---

# String Immutability

Strings cannot be modified directly.

Wrong:

```python
hostname = "web01"

hostname[0] = "W"
```

Error:

```text
TypeError
```

Strings are immutable.

Create a new string instead.

```python
hostname = "Web01"
```

---

# String Methods

Methods perform operations on strings.

---

## upper()

Convert to uppercase.

```python
environment = "production"

print(environment.upper())
```

Output:

```text
PRODUCTION
```

---

## lower()

Convert to lowercase.

```python
server = "WEB01"

print(server.lower())
```

Output:

```text
web01
```

---

## title()

```python
message = "deployment completed"

print(message.title())
```

Output:

```text
Deployment Completed
```

---

## strip()

Removes extra spaces.

```python
server = "   web01   "

print(server.strip())
```

Output:

```text
web01
```

---

## replace()

```python
hostname = "web01"

print(hostname.replace("web", "api"))
```

Output:

```text
api01
```

---

# Modern String Formatting (f-Strings)

This is the preferred method in Python 3.14.

---

## Basic Example

```python
hostname = "web01"

print(f"Server: {hostname}")
```

Output:

```text
Server: web01
```

---

## Multiple Variables

```python
hostname = "web01"
environment = "production"

print(f"{hostname} is running in {environment}")
```

Output:

```text
web01 is running in production
```

# Debugging Lab

## Broken Code

```python
hostname = web01

print(hostname)
```

Error:

```text
NameError: name 'web01' is not defined
```

---

## Root Cause

Strings must be surrounded by quotes.

Python interprets:

```python
web01
```

as a variable name.

---

## Fix

```python
hostname = "web01"

print(hostname)
```

Output:

```text
web01
```

---

## Explanation

Text values must always be enclosed in quotation marks.

# Key Takeaways

* Strings store text data.
* Strings are one of the most common data types in DevOps automation.
* Strings use quotation marks.

* Slicing extracts portions of text.
* Strings are immutable.
* Methods allow string transformation.
* f-strings are the modern way to format text.
* Infrastructure reports, logs, and alerts rely heavily on strings.
* Strong string skills are essential for automation work.
