# Module 10 — Loops (for and while)

---

# Why This Matters

Imagine you are responsible for:

```text
5 servers
```

You need to check each server.

Without loops:

```python
print("Checking web01")
print("Checking web02")
print("Checking web03")
print("Checking web04")
print("Checking web05")
```

This works.

But what happens when the company grows?

```text
50 servers
500 servers
5000 servers
```

Will you write:

```python
print(...)
print(...)
print(...)
```

thousands of times?

Of course not.

This is exactly why loops exist.

A loop allows Python to repeat work automatically.

Instead of writing code multiple times:

```text
Write Once
Run Many Times
```

This idea is one of the foundations of automation.

As a DevOps Engineer, loops are everywhere:

* Checking server inventories
* Processing log files
* Monitoring infrastructure
* Deploying applications
* Running backups
* Scanning directories
* Reading configuration files
* Calling APIs repeatedly

Without loops, automation would not exist.

---

Python performed:

```text

Run Code

Take Third Server
Run Code
```

Automatically.

---

Visual:

```text
servers

web01
web02
web03

     ↓

Loop

     ↓

web01
web02
web03
```

# Core Concepts

---

# What Is a Loop?

A loop repeatedly executes a block of code.

General idea:

```text
Repeat
Repeat
Repeat
Repeat
Until Finished
```

---

Python provides two main loop types:

```text
for loop
while loop
```

---

# Part 1 — for Loops

---

# Why for Loops Exist

Suppose:

```python
servers = [
    "web01",
    "web02",
    "web03"
]
```

Instead of:

```python
print(servers[0])
print(servers[1])
print(servers[2])
```

Use:

```python
for server in servers:
    print(server)
```

Cleaner.

More scalable.

More professional.

---

# Basic for Loop Syntax

```python
for item in collection:
    code
```

---

Example:

```python

    "web01",
    "web02",
    "web03"
]

for server in servers:
    print(server)
```

Output:

```text
web01
web02
web03
```

# Understanding Loop Variables

Very important for DevOps.

# Part 2 — while Loops

---

# What Is a while Loop?

A while loop repeats while a condition remains True.

Syntax:

```python
while condition:
    code
```

---

Example:

```python
count = 1

while count <= 5:
    print(count)

    count += 1
```

Output:

```text
1
2
3
4
5
```

---

Visual:

```text
count = 1

1 <= 5

YES

Run Loop

count = 2

2 <= 5

YES

Run Loop

...

count = 6

6 <= 5

NO

Stop
```

---

# Infinite Loops

Dangerous.

Example:

```python
while True:
    print("Running")
```

This never stops.

---

Visual:

```text
True

↓

Loop

↓

True

↓

Loop

↓

Forever
```

---

In production systems, infinite loops must be carefully controlled.

---

# break Statement

Used to exit a loop early.

Example:

```python
for number in range(10):

    if number == 5:
        break

    print(number)
```

Output:

```text
0
1
2
3
4
```

---

Visual:

```text
0
1
2
3
4

Encounter 5

STOP
```

---

# continue Statement

Skip current iteration.

Example:

```python
for number in range(5):

    if number == 2:
        continue

    print(number)
```

Output:

```text
0
1
3
4
```

---

Visual:

```text
0
1

Skip 2

3
4
```

# Nested Loops

A loop inside another loop.

Example:

```python
environments = [
    "dev",
    "prod"
]

servers = [
    "web01",
    "web02"
]

for env in environments:

    for server in servers:
        print(env, server)
```

Output:

```text
dev web01
dev web02
prod web01
prod web02
```

---

Visual:

```text
dev
 ├─ web01
 └─ web02

prod
 ├─ web01
 └─ web02
```

# Debugging Lab

## Broken Code

```python
servers = [
    "web01",
    "web02"
]

for server in servers
    print(server)
```

Error:

```text
SyntaxError
```

---

## Root Cause

Missing colon.

Wrong:

```python
for server in servers
```

Correct:

```python
for server in servers:
```

---

## Fixed Code

```python
servers = [
    "web01",
    "web02"
]

for server in servers:
    print(server)
```

Output:

```text
web01
web02
```

# Common Beginner Mistakes

## Mistake 1

Infinite while loop.

Wrong:

```python
count = 1

while count <= 5:
    print(count)
```

Problem:

```text
count never changes
```

---

Correct:

```python
count = 1

while count <= 5:

    print(count)

    count += 1
```

---

## Mistake 2

Using wrong indentation.

Wrong:

```python
for item in items:
print(item)
```

---

Correct:

```python
for item in items:
    print(item)
```

---

## Mistake 3

Misunderstanding range.

Wrong expectation:

```python
range(5)
```

Output:

```text
0
1
2
3
4
5
```

Actual:

```text
0
1
2
3
4
```

# Best Practices

## Prefer for Loops When Iterating Collections

Good:

```python
for server in servers:
```

---

## Use Meaningful Variable Names

Good:

```python
for server in servers:
```

Bad:

```python
for x in servers:
```

---

## Avoid Infinite Loops

Always ensure:

```python
while
```

conditions eventually become False.

---

## Keep Loop Bodies Small

Readable loops are easier to maintain.

---

## Use Loops To Eliminate Repetition

If you copy and paste code repeatedly:

```text
A loop is probably needed.
```

# Key Takeaways

* Loops allow code to run repeatedly.
* `for` loops iterate through collections.
* `while` loops repeat while a condition is True.
* `range()` generates sequences of numbers.
* `break` exits a loop early.
* `continue` skips the current iteration.
* Dictionaries can be processed with loops.
* Nested loops handle multiple dimensions of data.
* Loops are the foundation of automation.
* Most DevOps scripts rely heavily on loops.
