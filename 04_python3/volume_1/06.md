# Module 06 — Dictionaries

---

# Why This Matters

In the previous module, we learned how lists can store multiple values.

Example:

```python
servers = [
    "web01",
    "web02",
    "web03"
]
```

This works well when all we need is a collection of values.

But real infrastructure contains more information.

Consider a production server:

```text
Hostname: web01
IP Address: 10.0.0.5
Environment: production
CPU Cores: 8
Memory: 32 GB
Status: healthy
```

Can a list store this?

```python
server = [
    "web01",
    "10.0.0.5",
    "production",
    8,
    32,
    "healthy"
]
```

Technically yes.

But now a problem appears.

What does:

```python
server[3]
```

mean?

CPU?

Memory?

Status?

Nobody knows without reading the entire list structure.

This quickly becomes difficult to maintain.

Python provides a much better solution:

**Dictionaries**

Dictionaries allow us to store data using meaningful labels.

Instead of:

```python
server[3]
```

we can write:

```python
server["cpu_cores"]
```

Much clearer.

Much safer.

Much more maintainable.

As a DevOps Engineer, dictionaries are everywhere:

* API responses
* JSON data
* Kubernetes manifests
* Configuration files
* Infrastructure metadata
* Monitoring systems
* Cloud automation
* Deployment pipelines

If lists are one of the most important Python data structures, dictionaries are arguably the most important.

---

# Quick Example

```python
server = {
    "hostname": "web01",
    "ip_address": "10.0.0.5",
    "environment": "production"
}

print(server["hostname"])
```

Output:

```text
web01
```

---

## What Happened?

Python created a dictionary.

Visual representation:

```text
server

hostname     → web01
ip_address   → 10.0.0.5
environment  → production
```

Instead of using indexes:

```python
server[0]
```

we use meaningful names:

```python
server["hostname"]
```

---

# Core Concepts

---

## What Is a Dictionary?

A dictionary is a collection of:

```text
Key → Value
```

pairs.

Example:

```python
server = {
    "hostname": "web01",
    "environment": "production"
}
```

Visual:

```text
hostname
    ↓
 web01

environment
    ↓
 production
```

---

## Why Dictionaries Exist

Lists answer:

```text
"What is item number 3?"
```

Dictionaries answer:

```text
"What is the hostname?"
```

For infrastructure data, the second question is far more useful.

---

## Dictionary Terminology

Example:

```python
server = {
    "hostname": "web01"
}
```

Key:

```python
"hostname"
```

Value:

```python
"web01"
```

Together:

```text
hostname → web01
```

---

# Dictionary Syntax

Dictionaries use curly braces.

```python
server = {
    "hostname": "web01",
    "ip_address": "10.0.0.5",
    "environment": "production"
}
```

General form:

```python
dictionary = {
    key: value,
    key: value
}
```

# Updating Values

Dictionaries are mutable.

Example:

```python
server = {
    "environment": "development"
}

server["environment"] = "production"

print(server)
```

Output:

```text
{'environment': 'production'}
```

---

# Adding New Keys

```python
server = {
    "hostname": "web01"
}

server["ip_address"] = "10.0.0.5"

print(server)
```

Output:

```text
{
 'hostname': 'web01',
 'ip_address': '10.0.0.5'
}
```

---

# Removing Keys

Use:

```python
pop()
```

Example:

```python
server = {
    "hostname": "web01",
    "environment": "production"
}

server.pop("environment")

print(server)
```

Output:

```text
{'hostname': 'web01'}
```

---

# Dictionary Length

Use:

```python
len()
```

Example:

```python
server = {
    "hostname": "web01",
    "ip_address": "10.0.0.5",
    "environment": "production"
}

print(len(server))
```

Output:

```text
3
```

---

# Getting All Keys

Use:

```python
keys()
```

Example:

```python
server = {
    "hostname": "web01",
    "ip_address": "10.0.0.5"
}

print(server.keys())
```

Output:

```text
dict_keys(['hostname', 'ip_address'])
```

---

# Getting All Values

Use:

```python
values()
```

Example:

```python
print(server.values())
```

Output:

```text
dict_values(['web01', '10.0.0.5'])
```

---

# Getting Both Keys and Values

Use:

```python
items()
```

Example:

```python
print(server.items())
```

Output:

```text
dict_items([
 ('hostname', 'web01'),
 ('ip_address', '10.0.0.5')
])
```

---

# Safe Access with get()

Suppose:

```python
server = {
    "hostname": "web01"
}
```

Wrong:

```python
print(server["status"])
```

Error:

```text
KeyError
```

Safer:

```python
print(server.get("status"))
```

Output:

```text
None
```

---

Even better:

```python
print(server.get("status", "unknown"))
```

Output:

```text
unknown
```

# Nested Dictionaries

Dictionaries can contain dictionaries.

```python
infrastructure = {
    "web_server": {
        "hostname": "web01",
        "ip": "10.0.0.5"
    }
}
```

Access nested values:

```python
print(
    infrastructure["web_server"]["hostname"]
)
```

Output:

```text
web01
```

---

This structure is extremely common in:

```text
JSON
APIs
Cloud Automation
Configuration Files
```

We will revisit it many times later.

# Debugging Lab

## Broken Code

```python
server = {
    "hostname": "web01"
}

print(server["status"])
```

Error:

```text
KeyError: 'status'
```

---

## Root Cause

The key does not exist.

Available keys:

```text
hostname
```

Requested key:

```text
status
```

---

## Fix

```python
server = {
    "hostname": "web01"
}

print(
    server.get("status", "unknown")
)
```

Output:

```text
unknown
```

---

## Explanation

`get()` prevents crashes when keys are missing.

# Common Beginner Mistakes

## Mistake 1

Wrong

```python

    "hostname": "web01"
]
```

Correct

```python
server = {
    "hostname": "web01"
}
```

Dictionaries use curly braces.

---

## Mistake 2

Wrong

```python
server = {
    hostname: "web01"
}
```

Correct

```python
server = {
    "hostname": "web01"
}
```

Keys should usually be strings.

---

## Mistake 3

Wrong

```python
server["status"]
```

when the key does not exist.

Correct

```python
server.get("status")
```

# Best Practices

## Use Meaningful Keys

Good:

```python
"hostname"
```

Bad:

```python
"h"
```

---

## Keep Key Names Consistent

Good:

```python
hostname
ip_address
cpu_cores
memory_gb
```

Avoid mixing naming styles.

---

## Prefer get() For Optional Data

Good:

```python
server.get("owner")
```

Safer than direct indexing.

---

## Use Dictionaries for Structured Data

If data has labels, use a dictionary.

---

## Design for Readability

Infrastructure automation is maintained for years.

Choose names future engineers can understand immediately.

# Key Takeaways

* Dictionaries store data using key-value pairs.
* Keys provide meaningful labels for values.
* Dictionaries are ideal for structured infrastructure data.
* Values are accessed using keys.
* Dictionaries are mutable.
* `get()` provides safe lookups.
* `keys()`, `values()`, and `items()` help inspect data.
* Nested dictionaries are common in APIs and JSON.
* Dictionaries are one of the most important tools in Python automation.
* Most DevOps configuration and API data eventually becomes Python dictionaries.
