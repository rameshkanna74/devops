# Module 09 — Conditional Logic (if, elif, else)

---

# Why This Matters

In Module 08, we learned how to ask questions.

Example:

```python
cpu_usage = 92

print(cpu_usage > 90)
```

Output:

```text
True
```

This is useful.

But what should happen next?

Suppose a monitoring system detects:

```text
CPU Usage: 92%
```

Should the system:

```text
Send Alert?
Restart Service?
Scale Infrastructure?
Ignore?
```

The answer depends on a decision.

Programs become useful when they can react differently based on conditions.

This ability is called:

```text
Conditional Logic
```

Conditional logic allows programs to make decisions.

Examples:

```text
If CPU > 90%, send alert

If deployment succeeded, continue

If deployment failed, stop

If backup exists, skip backup

If disk space is low, notify administrator
```

Almost every automation script contains conditional logic.

Without it, programs would execute the same actions every time.

With it, programs become intelligent.

# Core Concepts

---

# What Is Conditional Logic?

Conditional logic allows programs to choose different actions based on conditions.

General idea:

```text
IF something is true
    do something
```

Example:

```text
IF server is offline
    send alert
```

---

# The if Statement

Basic syntax:

```python
if condition:
    code
```

Example:

```python
server_online = True

if server_online:
    print("Server is reachable")
```

Output:

```text
Server is reachable
```

---

## Important Rule

The colon:

```python
:
```

is mandatory.

Correct:

```python
if True:
    print("Hello")
```

Wrong:

```python
if True
    print("Hello")
```

SyntaxError.

---

## Indentation Matters

Python uses indentation to define blocks.

Correct:

```python
if True:
    print("Running")
```

Wrong:

```python
if True:
print("Running")
```

Error:

```text
IndentationError
```

---

# if Statement Flow

Example:

```python
cpu_usage = 95

if cpu_usage > 90:
    print("Critical Alert")
```

Flow:

```text
95 > 90

↓

True

↓

Run Code
```

---

If condition is False:

```python
cpu_usage = 50

if cpu_usage > 90:
    print("Critical Alert")
```

Output:

```text
(no output)
```

Because:

```text
50 > 90

↓

False

↓

Skip Code
```

---

# The else Statement

Sometimes we want another action when the condition is False.

Syntax:

```python
if condition:
    code
else:
    code
```

Example:

```python
cpu_usage = 50

if cpu_usage > 90:
    print("Critical Alert")
else:
    print("CPU Normal")
```

Output:

```text
CPU Normal
```

---

Visual:

```text
Condition?

├── True
│     Run IF Block
│
└── False
      Run ELSE Block
```

---

# The elif Statement

Sometimes two choices are not enough.

Example:

```text
Healthy
Warning
Critical
```

Python provides:

```python
elif
```

meaning:

```text
Else If
```

---

Syntax:

```python
if condition:
    code

elif condition:
    code

else:
    code
```

---

Example:

```python
cpu_usage = 75

if cpu_usage > 90:
    print("Critical")

elif cpu_usage > 70:
    print("Warning")

else:
    print("Healthy")
```

Output:

```text
Warning
```

---

Flow:

```text
75 > 90 ?

No

↓

75 > 70 ?

Yes

↓

Warning
```

---

# Multiple Conditions

Python evaluates conditions from top to bottom.

Example:

```python
cpu_usage = 95

if cpu_usage > 90:
    print("Critical")

elif cpu_usage > 70:
    print("Warning")

else:
    print("Healthy")
```

Output:

```text
Critical
```

Once Python finds a matching condition:

```text
It stops checking.
```

# Nested Conditions

An if statement can contain another if statement.

Example:

```python
environment = "production"
approved = True

if environment == "production":

    if approved:
        print("Deploy")

    else:
        print("Approval Required")
```

Output:

```text
Deploy
```

---

Visual:

```text
Production?

Yes

↓

Approved?

Yes

↓

Deploy
```

# Truthy and Falsy Conditions

Python allows:

```python
if value:
```

instead of:

```python
if value == True:
```

Example:

```python
server_online = True

if server_online:
    print("Online")
```

Preferred style.

# Best Practices

## Use Meaningful Conditions

Good:

```python
cpu_usage > 80
```

Bad:

```python
cpu_usage > 83.72
```

unless justified.

---

## Keep Conditions Simple

Good:

```python
environment == "production"
```

Easy to understand.

---

## Order Conditions Carefully

Most critical conditions should appear first.

Example:

```python
if cpu_usage > 90:
```

before:

```python
elif cpu_usage > 70:
```

---

## Use Boolean Variables

Good:

```python
deployment_successful
backup_exists
server_online
```

---

## Write Conditions Like English

Example:

```python
if backup_exists:
```

reads naturally.

# Key Takeaways

* `if` executes code when a condition is True.
* `else` executes when the condition is False.
* `elif` allows multiple decision paths.
* Conditions are evaluated from top to bottom.
* Python stops after the first matching condition.
* Indentation defines code blocks.
* Conditional logic powers monitoring and automation.
* Real DevOps systems rely heavily on decisions and validations.
* Booleans and comparisons drive conditional execution.
* Conditional logic transforms programs from static scripts into intelligent automation tools.
