# Module 08 — Booleans and Comparison Operators

---

# Why This Matters

So far, our programs can:

* Store information
* Perform calculations
* Work with text
* Manage collections of data

But they cannot make decisions.

Imagine a monitoring system.

The server reports:

```text
CPU Usage: 92%
```

How should the program respond?

Should it:

```text
Send Alert?
Restart Service?
Scale Infrastructure?
Ignore?
```

The answer depends on a condition.

Programs make decisions by asking questions.

Examples:

```text
Is CPU usage above 90%?
Is disk space below 10%?
Is the server online?
Is the deployment successful?
Does the backup exist?
```

Every one of these questions has only two possible answers:

```text
Yes
No
```

In Python:

```text
True
False
```

These values are called **Booleans**.

Booleans and comparison operators are the foundation of:

* Monitoring systems
* Deployment validation
* Health checks
* Security automation
* Cloud automation
* Infrastructure management

Without them, programs cannot make intelligent decisions.

# What Is a Boolean?

A Boolean is a value that represents truth.

Python has exactly two Boolean values:

```python
True
False
```

Notice the capitalization.

Correct:

```python
True
False
```

Wrong:

```python
true
false
```

---

## Why Booleans Exist

Computers make decisions using Boolean values.

Example:

```text
Server Healthy?

YES
↓
True
```

```text
Disk Full?

NO
↓
False
```

# Comparison Operators

Comparison operators compare values.

The result is always:

```python
True
```

or

```python
False
```

---

| Operator | Meaning                  |
| -------- | ------------------------ |
| ==       | Equal To                 |
| !=       | Not Equal To             |
| >        | Greater Than             |
| <        | Less Than                |
| >=       | Greater Than or Equal To |
| <=       | Less Than or Equal To    |

# Truthy and Falsy Values

Python treats some values as False.

Examples:

```python

0
""
[]
{}

```

---

Everything else is generally considered True.

Example:

```python
print(bool(0))
```

Output:

```text
False
```

---

```python
print(bool(1))
```

Output:

```text
True
```

---

```python
print(bool("hello"))
```

Output:

```text
True
```

---

# Debugging Lab

## Broken Code

```python
cpu_usage = 95

print(cpu_usage = 90)
```

Error:

```text
SyntaxError
```

---

## Root Cause

Single equals:

```python
=
```

means assignment.

Comparison requires:

```python
==
```

---

## Fix

```python
cpu_usage = 95

print(cpu_usage == 90)
```

Output:

```text
False
```

---

## Explanation

Remember:

```text
=   → Assign Value
==  → Compare Values
```

This is one of the most common beginner mistakes.

# Best Practices

## Use Descriptive Boolean Names

Good:

```python
server_online
backup_completed
deployment_successful
```

Bad:

```python
x
flag
```

---

## Write Comparisons Clearly

Good:

```python
cpu_usage > 90
```

Avoid overly complex expressions.

---

## Use Meaningful Thresholds

Good:

```python
cpu_usage > 80
```

Bad:

```python
cpu_usage > 83.274
```

unless there is a specific reason.

---

## Keep Boolean Variables Readable

Good:

```python
is_healthy
has_backup
is_reachable
```

These read like natural language.

# Key Takeaways

* Booleans represent truth values.
* Python has only two Boolean values: `True` and `False`.
* Comparison operators compare values.
* Every comparison returns a Boolean result.
* `=` assigns values.
* `==` compares values.
* Membership testing uses `in`.
* Strings can be compared.
* Monitoring systems rely heavily on Boolean logic.
* Booleans are the foundation of decision-making in Python.
