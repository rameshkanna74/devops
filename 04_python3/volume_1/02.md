# Module 02 — Variables and Data

---

# Why This Matters

Imagine you are managing a production server.

The server has information such as:

* Hostname
* IP Address
* Environment
* CPU Usage
* Available Memory

Your programs need a way to remember this information.

Without variables, every piece of information would have to be typed repeatedly throughout your code.

Variables allow Python programs to store information so it can be used later.

Think of a variable as a labeled storage box.

Example:

```text
Hostname Box
     |
     v
"web01"
```

Instead of remembering the actual value, your program remembers the label.

This makes programs easier to read, maintain, and automate.

Variables are the foundation of programming.

Everything you build later in this handbook—automation scripts, monitoring tools, Docker utilities, API clients, and cloud automation—will rely heavily on variables.

---

## Real DevOps Examples

Variables are used everywhere:

```python
server_name = "web01"
```

```python
environment = "production"
```

```python
cpu_usage = 73
```

```python
backup_enabled = True
```

Without variables, automation would be nearly impossible.

# Core Concepts

---

## What Is a Variable?

A variable is a name that refers to a value.

Example:

```python
server_name = "web01"
```

Visual representation:

```text
server_name
     |
     v
   "web01"
```

---

## Why Variables Exist

Variables solve a simple problem:

Instead of writing values repeatedly:

```python
print("web01")
print("web01")
print("web01")
```

Store once:

```python
server_name = "web01"

print(server_name)
print(server_name)
print(server_name)
```

This makes updates easier.

---

## Assignment Operator

The equals sign:

```python
=
```

is called the assignment operator.

Example:

```python
server_name = "web01"
```

Meaning:

```text
Store "web01"
inside server_name
```

It does NOT mean:

```text
equals in mathematics
```

---

## Variables Can Change

Variables are called variables because their values can vary.

```python
environment = "development"

print(environment)

environment = "production"

print(environment)
```

Output:

```text
development
production
```

---

## Python Is Dynamically Typed

Python automatically determines the data type.

Example:

```python
server_count = 10
```

Python knows:

```text
10 → integer
```

Example:

```python
server_name = "web01"
```

Python knows:

```text
web01 → string
```

You do not need to manually specify types.

# Variable Naming Rules

Good names improve readability.

---

## Valid Names

```python
server_name = "web01"

cpu_usage = 72

memory_available = 32
```

---

## Invalid Names

Cannot start with a number.

Wrong:

```python
1server = "web01"
```

---

Cannot contain spaces.

Wrong:

```python
server name = "web01"
```

---

Cannot use Python keywords.

Wrong:

```python
class = "web01"
```

---

# Naming Best Practices

Use descriptive names.

Good:

```python
database_server_name = "db01"
```

Bad:

```python
x = "db01"
```

---

Use snake_case.

Good:

```python
backup_directory
```

Bad:

```python
backupDirectory
```

For this handbook we will follow:

```text
snake_case
```

everywhere.

# Debugging Lab

## Broken Code

```python
server name = "web01"
```

Error:

```text
SyntaxError
```

---

## Root Cause

Variable names cannot contain spaces.

Python interprets:

```text
server
name
```

as separate tokens.

---

## Fix

```python
server_name = "web01"
```

Output:

```text
web01
```

---

## Explanation

Always use:

```text
snake_case
```

for multi-word variable names.
