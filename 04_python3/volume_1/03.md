# Quick Example

Suppose a server has:

```text
8 CPU cores
```

and you add:

```text
4 CPU cores
```

Python can calculate the total.

```python
current_cpu = 8
additional_cpu = 4

total_cpu = current_cpu + additional_cpu

print(total_cpu)
```

Output:

```text
12
```

---

## What Happened?

Python performed:

```text
8 + 4
```

and stored the result inside:

```python
total_cpu
```

Variables can store not only values but also calculation results.

Examples:

```python
10

```

These are numbers.

CPU Cores

## Python Number Types

Python has multiple numeric types.

For beginners, focus on two.

| Type  | Description    | Example |
| ----- | -------------- | ------- |
| int   | Whole Number   | 10      |
| float | Decimal Number | 10.5    |

---

## Integer (int)

Integers have no decimal point.

Examples:

```python
server_count = 12

port = 443

cpu_cores = 8
```

---

## Float

Floats contain decimal values.

Examples:

```python
cpu_usage = 82.5

memory_usage = 73.2

disk_usage = 91.8
```

---

## Checking Types

Use:

```python

```

Example:

```python

```

```text

```

---

Example:

```python
cpu_usage = 82.5

print(type(cpu_usage))
```

Output:

```text

```

# Arithmetic Operators

Python supports standard mathematical operations.

| Operator | Meaning        |
| -------- | -------------- |
| +        | Addition       |
| -        | Subtraction    |
| *        | Multiplication |
| /        | Division       |
| //       | Floor Division |
| %        | Modulo         |
| **       | Exponent       |

---

# Addition

```python

```

Output:

```text
12
```

---

# Subtraction

```python

```

Output:

```text
46
```

---

# Multiplication

```python
server_count = 5
cpu_per_server = 4

print(server_count * cpu_per_server)
```

Output:

```text
20
```

---

# Division

```python
total_storage = 100
servers = 4

print(total_storage / servers)
```

Output:

```text

```

Notice:

```text
Division returns a float.
```

---

# Floor Division

Removes decimal places.

```python
print(10 // 3)
```

Output:

```text
3
```

---

# Modulo

Returns the remainder.

```python
print(10 % 3)
```

Output:

```text
1
```

Visualization:

```text
10 ÷ 3

3 remainder 1
```

---

# Exponent

Power calculations.

```python
print(2 ** 3)
```

Output:

```text
8
```

Because:

```text
2 × 2 × 2 = 8
```

---

# Order of Operations

Python follows mathematical precedence.

Example:

```python
print(2 + 3 * 4)
```

Output:

```text
14
```

Python evaluates:

```text
3 * 4 = 12

2 + 12 = 14
```

---

Use parentheses to control execution.

```python
print((2 + 3) * 4)
```

Output:

```text
20
```

---

# Syntax

Basic calculations:

```python

result = 3 * 4

```

# Floating Point Precision

Computers store decimals differently than humans.

Example:

```python
print(0.1 + 0.2)
```

Output:

```text
0.30000000000000004
```

This is normal.

It is not a bug.

Most infrastructure automation does not require concern about this issue.

# Debugging Lab

## Broken Code

```python
cpu_cores = 8

print(cpu_core)
```

Error:

```text
NameError: name 'cpu_core' is not defined
```

---

## Root Cause

Variable name mismatch.

Defined:

```python
cpu_cores
```

Used:

```python
cpu_core
```

Python treats them as different names.

---

## Fix

```python
cpu_cores = 8

print(cpu_cores)
```

Output:

```text
8
```

---

## Explanation

Variable names must match exactly.

Python is case-sensitive.

# Common Beginner Mistakes

## Mistake 1

Wrong:

```python
memory = "64"

result = memory + 10
```

Problem:

```text
String and integer cannot be added.
```

Correct:

```python
memory = 64

result = memory + 10
```

---

## Mistake 2

Wrong:

```python
print(2 + 3 * 4)
```

Expecting:

```text
20
```

Actual:

```text
14
```

Correct:

```python
print((2 + 3) * 4)
```

---

## Mistake 3

Wrong:

```python
print(10 / 0)
```

Error:

```text
ZeroDivisionError
```

Always ensure divisors are not zero.

# Best Practices

Good:

```python

```

```python

```

---

## Store Intermediate Results

Good:

```python
available_memory = total_memory - used_memory
```

Instead of:

```python

```

---

## Use Parentheses

Improve readability.

```python
cpu_percent = (used_cores / total_cores) * 100
```

---

## Prefer Clarity Over Cleverness

Write code that teammates can understand.

## Verify Results

When automating infrastructure calculations, always validate outputs.
