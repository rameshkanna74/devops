# Module 07 — Tuples and Sets

---

# Why This Matters

In the previous modules, we learned about:

* Variables
* Numbers
* Strings
* Lists
* Dictionaries

Lists are incredibly useful because they allow us to store multiple values.

However, not all data should be changeable.

Consider:

```text
Production Database Port: 5432
SSH Port: 22
HTTPS Port: 443
```

These values rarely change.

If a developer accidentally modifies them, automation scripts may break.

Similarly, consider a list of server IP addresses:

```text
10.0.0.5
10.0.0.6
10.0.0.5
10.0.0.7
10.0.0.6
```

Notice the duplicates.

Many infrastructure tasks require:

* Removing duplicates
* Fast lookups
* Tracking unique values

Python provides two specialized data structures:

```text
Tuple → Immutable Collection
Set   → Unique Collection
```

Understanding when to use these structures will help you write cleaner, safer, and faster automation scripts.

---

# Quick Example

## Tuple Example

```python
ports = (22, 80, 443)

print(ports)
```

Output:

```text
(22, 80, 443)
```

---

servers = {

# Core Concepts

---

# Part 1 — Tuples

---

## What Is a Tuple?

A tuple is an ordered collection of values that cannot be modified after creation.

Think of a tuple as:

```text
Read-Only List
```

Example:

```python
ports = (22, 80, 443)
```

---

## Why Tuples Exist

Sometimes data should not change.

Examples:

```text
Country Codes
Port Numbers
API Endpoints
Supported Protocols
Configuration Constants
```

Using tuples helps prevent accidental modifications.

---

## Tuple vs List

List:

```python
servers = ["web01", "web02"]
```

Tuple:

```python
servers = ("web01", "web02")
```

Difference:

```text
List  → Mutable
Tuple → Immutable
```

---

## Creating Tuples

### Multiple Items

```python
ports = (22, 80, 443)

print(ports)
```

Output:

```text
(22, 80, 443)
```

---

### String Tuple

```python
environments = (
    "development",
    "testing",
    "production"
)

print(environments)
```

Output:

```text
('development', 'testing', 'production')
```

---

### Single Item Tuple

Special syntax:

```python
port = (22,)
```

Without comma:

```python
port = (22)
```

This becomes:

```python
int
```

not a tuple.

---

## Accessing Tuple Values

Uses indexing just like lists.

```python
ports = (22, 80, 443)

print(ports[0])
```

Output:

```text
22
```

---

```python
print(ports[2])
```

Output:

```text
443
```

---

## Tuple Length

```python
ports = (22, 80, 443)

print(len(ports))
```

Output:

```text
3
```

---

## Tuple Slicing

```python
ports = (22, 80, 443, 8080)

print(ports[1:3])
```

Output:

```text
(80, 443)
```

---

## Tuple Immutability

This is the most important tuple feature.

Wrong:

```python
ports = (22, 80, 443)

ports[0] = 21
```

Error:

```text
TypeError
```

Because tuples cannot be modified.

# Part 2 — Sets

---

## What Is a Set?

A set is an unordered collection of unique values.

Key features:

```text
No Duplicates
Fast Lookups
Set Operations
```

---

## Why Sets Exist

Imagine processing logs.

You discover:

```text
web01
web02
web01
web03
web02
web01
```

You only want unique server names.

A set solves this automatically.

---

## Creating Sets

```python
servers = {
    "web01",
    "web02",
    "web03"
}

print(servers)
```

Output:

```text
{'web01', 'web02', 'web03'}
```

---

## Removing Duplicates

```python
servers = {
    "web01",
    "web02",
    "web01",
    "web03"
}

print(servers)
```

Output:

```text
{'web01', 'web02', 'web03'}
```

Duplicate values disappear automatically.

---

## Creating Set From List

Very common.

```python
servers = [
    "web01",
    "web02",
    "web01",
    "web03"
]

unique_servers = set(servers)

print(unique_servers)
```

Output:

```text
{'web01', 'web02', 'web03'}
```

---

## Adding Values

```python
servers = {"web01", "web02"}

servers.add("web03")

print(servers)
```

Output:

```text
{'web01', 'web02', 'web03'}
```

---

## Removing Values

```python
servers = {"web01", "web02"}

servers.remove("web02")

print(servers)
```

Output:

```text
{'web01'}
```

---

## Membership Testing

One of the biggest advantages of sets.

```python
servers = {
    "web01",
    "web02",
    "web03"
}

print("web02" in servers)
```

Output:

```text
True
```

---

```python
print("db01" in servers)
```

Output:

```text
False
```

---

# Set Operations

These become extremely useful later.

---

## Union

Combine sets.

```python
web_servers = {
    "web01",
    "web02"
}

api_servers = {
    "api01",
    "api02"
}

print(web_servers | api_servers)
```

Output:

```text
{'web01', 'web02', 'api01', 'api02'}
```

---

## Intersection

Find common values.

```python
cluster_a = {
    "web01",
    "web02"
}

cluster_b = {
    "web02",
    "web03"
}

print(cluster_a & cluster_b)
```

Output:

```text
{'web02'}
```

---

## Difference

Find values missing from another set.

```python
cluster_a = {
    "web01",
    "web02",
    "web03"
}

cluster_b = {
    "web02"
}

print(cluster_a - cluster_b)
```

Output:

```text
{'web01', 'web03'}
```

# Debugging Lab

## Broken Code

```python
ports = (22, 80, 443)

ports[0] = 21
```

Error:

```text
TypeError:
'tuple' object does not support item assignment
```

---

## Root Cause

Tuples are immutable.

Python prevents modification.

---

## Fix

Create a new tuple.

```python
ports = (21, 80, 443)

print(ports)
```

Output:

```text
(21, 80, 443)
```

---

## Explanation

Use tuples when values should remain constant.

Use lists when values must change.
