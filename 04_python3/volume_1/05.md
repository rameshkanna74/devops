# Module 05 — Lists

---

# Why This Matters

So far, our programs have stored one piece of information at a time.

Example:

```python
server_name = "web01"
```

This works when managing a single server.

But what happens when your infrastructure grows?

Imagine you manage:

```text
web01
web02
web03
api01
api02
db01
db02
```

Creating a separate variable for every server quickly becomes unmanageable.

```python
web_server_1 = "web01"
web_server_2 = "web02"
web_server_3 = "web03"
api_server_1 = "api01"
api_server_2 = "api02"
database_server_1 = "db01"
database_server_2 = "db02"
```

This approach does not scale.

Python provides a better solution:

**Lists**

A list allows us to store multiple related values inside a single variable.

Lists are one of the most important data structures in Python.

As a DevOps Engineer, you will use lists constantly:

* Server inventories
* IP addresses
* Docker containers
* Kubernetes pods
* Log files
* Backup targets
* Deployment environments
* Monitoring alerts

Almost every automation script uses lists.

Mastering lists is one of the biggest milestones in becoming a Python developer.

---

# Quick Example

```python
servers = ["web01", "web02", "web03"]

print(servers)
```

Output:

```text
['web01', 'web02', 'web03']
```

---

## What Happened?

Python created a list containing three values.

Visual representation:

```text
servers
    |
    v

+---------+
| web01   |
| web02   |
| web03   |
+---------+
```

Instead of creating three variables, we created one list.

# Core Concepts

---

## What Is a List?

A list is an ordered collection of items.

Example:

```python
servers = ["web01", "web02", "web03"]
```

The items remain in order.

```text
Position 0 → web01
Position 1 → web02
Position 2 → web03
```

---

## Why Lists Exist

Lists solve a common problem:

Store many related values together.

Instead of:

```python
server1 = "web01"
server2 = "web02"
server3 = "web03"
```

Use:

```python
servers = ["web01", "web02", "web03"]
```

Cleaner.

More scalable.

Much easier to automate.

---

## List Syntax

Lists use square brackets.

```python
servers = ["web01", "web02", "web03"]
```

General form:

```python
list_name = [item1, item2, item3]
```

---

## Lists Can Store Different Data Types

Although not always recommended.

```python
data = [
    "web01",
    80,
    True,
    4.5
]
```

Python allows this.

However, most production code stores related data together.

# Indexing

Lists use indexing.

First item:

```python
servers = ["web01", "web02", "web03"]

print(servers[0])
```

Output:

```text
web01
```

---

Visual:

```text
Index:

web01  web02  web03
  0      1      2
```

---

Second item:

```python
print(servers[1])
```

Output:

```text
web02
```

---

Third item:

```python
print(servers[2])
```

Output:

```text
web03
```

---

# Negative Indexing

Count backwards from the end.

```python
servers = ["web01", "web02", "web03"]

print(servers[-1])
```

Output:

```text
web03
```

---

Visual:

```text
web01  web02  web03
 -3     -2     -1
```

---

# List Length

Use:

```python
len()
```

Example:

```python
servers = ["web01", "web02", "web03"]

print(len(servers))
```

Output:

```text
3
```

---

# Slicing

Extract multiple items.

```python
servers = [
    "web01",
    "web02",
    "web03",
    "web04"
]

print(servers[0:2])
```

Output:

```text
['web01', 'web02']
```

---

Visual:

```text
0   1   2   3

web01
web02
web03
web04

0:2

web01
web02
```

---

# Modifying Lists

Unlike strings, lists are mutable.

---

## Change an Item

```python
servers = ["web01", "web02"]

servers[0] = "api01"

print(servers)
```

Output:

```text
['api01', 'web02']
```

---

# Adding Items

## append()

Adds to the end.

```python
servers = ["web01", "web02"]

servers.append("web03")

print(servers)
```

Output:

```text
['web01', 'web02', 'web03']
```

---

Visual:

```text
Before:

web01
web02

After:

web01
web02
web03
```

---

# Removing Items

## remove()

```python
servers = [
    "web01",
    "web02",
    "web03"
]

servers.remove("web02")

print(servers)
```

Output:

```text
['web01', 'web03']
```

---

# Sorting Lists

```python
servers = [
    "web03",
    "web01",
    "web02"
]

servers.sort()

print(servers)
```

Output:

```text
['web01', 'web02', 'web03']
```

---

# Reversing Lists

```python
servers = [
    "web01",
    "web02",
    "web03"
]

servers.reverse()

print(servers)
```

Output:

```text
['web03', 'web02', 'web01']
```

# Debugging Lab

## Broken Code

```python
servers = [
    "web01",
    "web02"
]

print(servers[5])
```

Error:

```text
IndexError: list index out of range
```

---

## Root Cause

The list contains only:

```text
0
1
```

Indexes.

There is no:

```text
5
```

---

## Fix

```python
print(servers[1])
```

Output:

```text
web02
```

---

## Explanation

Always verify the size of a list before accessing indexes.

Use:

```python
len()
```

when unsure.

# Common Beginner Mistakes

## Mistake 1

Wrong:

```python
servers = "web01", "web02"
```

Correct:

```python
servers = ["web01", "web02"]
```

Lists require square brackets.

---

## Mistake 2

Wrong:

```python
servers = ["web01"]

print(servers[1])
```

Error:

```text
IndexError
```

Correct:

```python
print(servers[0])
```

---

## Mistake 3

Wrong:

```python
servers = ["web01"]

servers.append(["web02"])
```

Output:

```text
['web01', ['web02']]
```

Sometimes beginners accidentally create nested lists.

Correct:

```python
servers.append("web02")
```

Output:

```text
['web01', 'web02']
```

# Best Practices

## Store Similar Data Together

Good:

```python
servers = [
    "web01",
    "web02",
    "web03"
]
```

Avoid mixing unrelated values.

---

## Use Meaningful Names

Good:

```python
deployment_targets
```

Bad:

```python
data
```

---

## Keep Lists Organized

Sort when appropriate.

```python
servers.sort()
```

---

## Prefer One Item Per Line

Large lists become easier to read.

```python
servers = [
    "web01",
    "web02",
    "web03"
]
```

---

## Avoid Magic Indexes

Bad:

```python
servers[4]
```

Good:

```python
primary_server = servers[0]
```

# Key Takeaways

* Lists store multiple values in a single variable.
* Lists maintain item order.
* Indexing starts at zero.
* Lists support slicing.
* Lists are mutable.
* `append()` adds items.
* `remove()` removes items.
* `sort()` organizes items.
* `len()` returns item count.
* Lists are one of the most important data structures in Python and DevOps automation.
